# Golden corpus

The cross-language golden: one blessed `EvolveReport` JSON per spec, regenerated
byte-for-byte by the Rust core and reproduced verbatim by every language binding.
Byte-identity holds because (1) the PRNG (SplitMix64) lives only in the Rust core
and (2) every binding returns the core's `command_json` string verbatim — no
per-language RNG, no JSON deep-equal, no float reformatting.

## Data (`data/*.csv`)

A fixed, reproducible universe of six symbols × 220 bars each. Each symbol's close
follows

```
close(i) = base + amp * sin(i / k) + drift * i,   i = 0 .. 219
open(i)  = close(i - 1)
high(i)  = max(open, close) + 1
low(i)   = min(open, close) - 1
volume   = 1000
time(i)  = 1_700_000_000 + i * 3600
```

with per-symbol constants (`base`, `amp`, `k`, `drift`):

| symbol  | base | amp | k  | drift |
|---------|------|-----|----|-------|
| sym-01  | 100  | 30  | 6  | 0.02  |
| sym-02  | 120  | 25  | 8  | 0.00  |
| sym-03  |  90  | 35  | 5  | 0.03  |
| sym-04  | 150  | 40  | 10 | 0.00  |
| sym-05  |  70  | 20  | 4  | 0.05  |
| sym-06  | 200  | 50  | 7  | 0.00  |

## Specs (`specs/*.json`)

Five canonical `EvolveSpec` files, each with a fixed `seed` that pins the entire
evolution:

- `evolve_small` — RSI, single-threshold rules, Sharpe fitness.
- `evolve_pnl` — EMA, single-threshold rules, PnL fitness.
- `evolve_calmar` — SMA, single-threshold rules, Calmar fitness.
- `evolve_crossover` — SMA/EMA, crossover-pair rules, Sharpe fitness.
- `evolve_conjunction` — RSI/ATR, conjunction-all rules (`max_conditions` 3), PnL fitness.

## Expected (`expected/*.json`)

One `EvolveReport` JSON per spec, generated by the Rust core and committed
byte-for-byte. **Never edit these by hand.** To re-bless after an intended change,
delete the file(s) and run the golden test — a missing expected file is written
from the current output:

```bash
rm golden/expected/<name>.json
cargo test -p darwin-core --test golden
```

The report's `history` array carries the per-generation `(best, mean, worst,
evaluated)` statistics; `best` is the deduplicated hall of fame of finite-fitness
genomes. On this smooth synthetic universe the randomly sampled strategies do not
clear the finite-fitness bar, so the hall of fame is empty and the per-generation
statistics are zero — the documented degenerate case (a non-trading spec scores
`NEG_INFINITY` and is filtered). The corpus still pins the exact report structure
and the seed-driven determinism that every binding must reproduce; richer trading
corpora are a fitness-tuning concern tracked separately.
